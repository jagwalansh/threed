threeD is an experimental no-code editor for designing animated 3D website experiences directly in the browser. It was made to make scroll-based 3D sections, motion paths, and interactive landing page visuals easier to create without requiring every designer or builder to set up a full Three.js project from scratch.

The idea behind threeD is simple: building 3D web experiences should feel closer to editing a visual canvas than hand-writing every scene, camera movement, object position, and animation step. Many modern websites use 3D product visuals, animated hero sections, and scroll-driven storytelling, but creating those experiences usually requires a mix of design tools, 3D knowledge, JavaScript, and a lot of manual integration work. threeD aims to bring those pieces together in one focused interface.

## What it provides

- A browser-based workspace for creating 3D website animation concepts.
- A visual editor for arranging sections, layers, motion paths, and scene settings.
- Controls for adjusting object position, rotation, scale, lighting, colors, and animation behavior.
- A preview canvas for testing how the experience feels while editing.
- Export-oriented code generation so the work can eventually move from prototype to production.
- A project home screen for starting and reopening recent 3D website animation projects.

## Why it was made

threeD was created to explore a faster workflow for building rich 3D web pages. Instead of starting with a blank codebase and manually wiring together every interaction, the goal is to provide a tool where creators can sketch, preview, and iterate visually first.

This project is especially useful for experimenting with:

- 3D animated landing pages.
- Scroll-driven website sections.
- Product-showcase concepts.
- Motion paths for 3D objects.
- No-code or low-code creative web tooling.
- Exportable Three.js or React-based website experiences.

## Project status

This project is not complete yet. It is still a prototype and there is more work to be done before it can be considered production-ready.

Some areas that still need improvement include:

- More reliable project saving and loading.
- Better export quality and framework support.
- More polished editor interactions.
- Improved asset handling.
- Better mobile and responsive behavior.
- More testing across browsers and devices.
- Documentation for setup, usage, and contribution.
